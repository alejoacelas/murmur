# Coverage note (SPEC §10.6 — what the green harness proves, and what it doesn't)

## Proven by the automated loop (`scripts/test.sh`)

- **Backend (layer 1):** real Parakeet model over committed fixtures; exact normalized match on short clips, WER = 0.0 on the 60 s clip (budget ≤ 0.1); silence → empty at both the VAD gate and the model.
- **Persistence/retry (layer 2):** 19 mock-backed tests — split states, transient auto-retry (≤3 + backoff), permanent no-thrash, insertFailed re-inserts without re-transcribing, crash-mid-recording finalization, orphan adoption, insertFailed left alone at launch.
- **Hotkey logic (layer 3):** 9 pure-matcher tests — latched modifiers, exact-match (Cmd+Ctrl+Space rejected), both edges swallowed, autorepeat no-flap, fn-space preset.
- **e2e (layer 4):** signed `Murmur.app` launched by exec → `wait-ready` → permissions asserted → fixtures injected → text pasted into focused `InsertionProbe.app` and read back; silence inserts nothing; zero error-level logs.
- **Clipboard restore (layer 5):** sentinel survives the paste cycle (documented lossy for custom providers).
- **Crash recovery (acceptance):** real SIGKILL mid-recording (realtime inject) and mid-transcription (fault delay), relaunch, session auto-completes from the saved CAF into the probe.
- **Hotkey smoke (layer 6, non-blocking):** synthetic Ctrl+Space through the real session CGEventTap starts/stops a REAL microphone session (VAD-gated). Runs in every e2e pass; verified synthetic modifier posts arrive as flagsChanged (hardware-identical to the latch).

## Covered only partially / not by the loop

- **Live microphone path:** `MicAudioSource` (hardware tap → convert → worker) is exercised by the layer-6 smoke (real `AVAudioEngine` capture of ambient audio) but its *transcript content* is never asserted — no virtual audio device (BlackHole) is installed to play fixtures into a mic. Assertion-grade mic coverage = install BlackHole (needs sudo) + play fixtures; declined as out of bootstrap scope.
- **HUD visuals:** phase/partial state is asserted via `murmurctl hud` + engine unit test; nobody looks at pixels. Panel behavior (non-activating, click-through, positioning) is by construction, verified once by eye… by nobody, actually — the human should glance at it on first real use.
- **Fn+Space:** never verified (Wispr Flow owns Fn on this machine; S6/S7 skipped). `fn-space` preset ships but is flagged unverified; Ctrl+Space is the default.
- **Multi-display/space edge cases, Secure-Input-mid-session, IME-heavy targets, screen-locked recovery pastes:** untested; failure mode is `insertFailed` (safe: transcript retained, manual retry).
- **Environment sensitivity:** the e2e needs the desktop to itself (probe must hold focus). The heartbeat/abort guard means a human touching the machine aborts the run rather than corrupting it — reruns are cheap.

## Notable non-obvious findings baked into the harness

- Cmd+V is a *menu* key equivalent: an app without an Edit menu silently ignores synthesized paste — InsertionProbe builds a real menu bar for this reason alone.
- Bare-binary self-activation is ignored under macOS 14+ cooperative activation when any other app holds focus; the probe must be launched via `open` (LaunchServices activation).
- `dispatchMain()` traps in Swift async top-level code; the app parks the real main thread in `RunLoop.main`/`NSApplication.run`.
- `sun_path` is ~104 bytes: control sockets can't live under deep temp roots; tests set `MURMUR_SOCK` to a short path.
- First model load *per binary content* pays a ~50–160 s CoreML/ANE compile (cache keys on the binary, not just the model); warm loads ~0.5 s. `wait-ready` timeouts sized accordingly.
